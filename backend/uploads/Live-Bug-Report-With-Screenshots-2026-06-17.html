API SECURITY RECOMMENDATIONS

These are backend-level observations. The frontend correctly blocks these actions via RBAC, so they are not counted as UI bugs per the testing mandate.

  1. Rate Limiting: No rate limiting observed on login endpoint — consider adding brute-force protection
  2. Session Management: Session tokens should be invalidated server-side on route errors (not just destroyed client-side)
  3. API Error Messages: Some API errors return stack traces in development mode — ensure production mode hides these
  4. CORS Policy: Verify CORS is configured to restrict origins in production
  5. Input Sanitization: Add server-side validation for timesheet descriptions to prevent XSS/injection
